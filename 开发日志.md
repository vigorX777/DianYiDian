## 2026-05-21 23:20 - MVP 初版落地

### 目标

根据 `MVP功能清单.md` 实现「点一点」菜单栏次数打卡工具的第一版可运行原型。

### 调整内容

- 创建 SwiftPM 工程，应用模块为 `DianYiDian`，核心业务模块为 `DianYiDianCore`。
- 实现单事项次数型打卡：默认事项“喝水”，默认目标 8 次，默认初始次数 0。
- 实现菜单栏常驻、左键加 1、右键菜单、设置窗口、本地存储和跨天归档。
- 增加本地 `.app` 打包脚本，生成 `点一点.app`。
- 增加核心业务校验入口 `DianYiDianCoreChecks`。

### 验证结果

- `swift build` 通过。
- `swift run DianYiDianCoreChecks` 通过。
- `sh Scripts/package-app.sh` 通过。
- 本地生成的 `点一点.app` 可以启动为进程。

### 发现的问题

- 当前机器没有完整 Xcode，暂时无法做标准 Xcode Archive。
- 首次验证时只确认进程启动，没有确认菜单栏项真实可见，验证标准不够完整。

### 下一步

- 增加针对菜单栏可见性的手动验收步骤。

## 2026-05-22 19:02 - 0.2.1 多场景缺陷修复

### 目标

修复多场景版本中的两个可见问题：菜单栏图标没有按设置页选择的 SF Symbols 正常展示；普通保存设置时反复提示开机自启更新失败。

### 调整内容

- `StatusIconRenderer` 统一使用 `IconStyle.symbolName` 渲染菜单栏和右键菜单场景图标。
- 移除水滴、圆点、对勾的旧自绘分支，避免设置页和菜单栏显示不一致。
- SF Symbols 图标改为基于 alpha mask 生成主题色符号，避免直接修改系统符号图像导致显示异常。
- `SettingsViewModel` 记录进入设置页时的开机自启状态，仅当开关发生变化时调用开机自启服务。
- `LaunchAtLoginService` 增加当前状态判断，未注册状态下关闭开机自启直接视为成功。
- 开机自启失败文案缩短，并且不影响其他场景设置保存。

### 验证结果

- `swift build` 通过。
- `swift run DianYiDianCoreChecks` 通过。
- `sh Scripts/package-app.sh` 通过。
- 启动 `点一点.app` 后，系统辅助功能可读到菜单栏项 `0/8, 0/1`。

### 发现的问题

- 图标是否完全符合预期仍需要在设置页切换具体图标后人工确认。
- SwiftPM 本地打包环境下，开机自启真实注册仍可能受签名或权限限制。

### 下一步

- 补充设置窗口和菜单栏图标的自动化截图验收能力。
