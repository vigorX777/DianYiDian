# 版本记录

## 0.4.6 - 提醒通道配置

### 新增

- 应用设置新增“系统通知”开关，可独立控制 macOS 本地通知提醒。
- 应用设置新增“菜单气泡”开关，可独立控制菜单栏轻提醒气泡。
- 设置页新增“开发者配置”，支持调整菜单栏轻提醒气泡显示秒数，默认 2 秒。

### 调整

- 场景级“菜单栏轻提示”保留，用于控制单个场景是否显示气泡。
- 系统通知关闭时不再请求通知权限，只保留菜单气泡通道。
- 气泡显示秒数限制为 0.5-10 秒，避免调试配置造成不可用体验。

## 0.4.5 - 轻提醒气泡交互修复

### 修复

- 修复没有独立菜单栏图标的场景触发轻提醒时，气泡缺少稳定锚点导致展示不稳定的问题。
- 轻提醒气泡显示时长调整为 2 秒。
- 轻提醒气泡改为可点击，点击后直接为对应场景完成一次打卡。
- 点击 macOS 系统通知横幅时，也会为对应场景完成一次打卡。

### 验证

- 复测新增场景提醒链路，确认未达标且开启提醒的场景可以触发通知。
- 确认已达标场景不会继续触发提醒，这是当前产品规则，不是通知异常。

## 0.4.4 - 间隔提醒稳定性修复

### 修复

- 修复间隔提醒只会在上次打卡后触发一次，之后不会继续提醒的问题。
- 间隔提醒现在按“上次打卡或上次提醒中较晚的时间 + 间隔分钟”计算下一次提醒。
- 将提醒定时器加入 main run loop 的 common mode，减少菜单交互期间定时器被暂停的情况。

### 验证

- 新增 CoreChecks 覆盖“刚提醒过暂不提醒”和“过了下一个间隔后再次提醒”。
- 使用 1 分钟间隔提醒做实机复现，确认上次提醒 70 秒后会再次触发通知。

## 0.4.3 - 提醒可见性修复

### 修复

- 设置 `UNUserNotificationCenterDelegate`，App 位于前台时本地通知也允许以横幅、列表和声音形式展示。
- 处理本地通知提交失败回调，失败时在设置页暴露简短错误提示。
- 修复授权刚完成后不会立即重新同步到期提醒的问题。

### 优化

- 到期提醒会同步触发菜单栏轻提示：对应场景图标短暂高亮，并弹出“该喝水了”这类短提示。
- 当系统通知未授权或被系统展示策略拦截时，仍保留菜单栏轻提示兜底。

## 0.4.2 - 菜单与提醒设置体验修复

### 修复

- 放大右键菜单日历日期格，修复日期数字视觉不居中的问题。
- 调整右键菜单玻璃底色透明度，降低背景过度模糊导致的可读性问题。
- 放大右键菜单底部操作按钮，提高点击面积。
- 场景切换行改为菜单内切换，并增加系统收起菜单后的自动重开兜底。
- 设置页右侧表单改为统一标签宽度和行间距，提升标题与控件的对齐一致性。

### 优化

- 间隔提醒支持手动输入任意大于 0 的分钟数。
- 固定时间提醒支持多个时间点，可逐项新增、删除和手动输入小时分钟。
- CoreChecks 增加任意间隔分钟和多固定时间提醒的校验。

## 0.4.1 - Liquid Glass 体验修复

### 修复

- 压缩右键玻璃 Popover 尺寸，避免顶部内容在菜单栏附近被裁切。
- 修复打卡后全量重建菜单栏图标导致多个场景同时出现响应的问题。

### 优化

- 提醒方式的间隔分钟、固定小时和固定分钟改为支持手动输入。
- 手动输入提醒时间时保留范围限制，保存前仍会做兜底校验。

## 0.4.0 - Liquid Glass UI 改造

### 新增

- 新增 `v0.4.0-Liquid-Glass-UI改造方案.md`，记录设计方向、范围、实施步骤和验收标准。

- 菜单栏图标增加玻璃底盘渐变、高光和更柔和的完成态质感。
- 达标小庆祝改为磨砂玻璃 popover 样式。

### 验证

- `sh Scripts/package-app.sh` 通过。
- 打包后的 `点一点.app` 可启动，菜单栏项可见。
- 左键打卡计数可更新。

- 设置窗口可打开并显示玻璃化界面。

## 0.3.0 - 轻提醒与打卡反馈

### 新增

- 每个场景新增提醒设置：不提醒、间隔提醒、固定时间提醒。
- 设置页新增“打卡动效”和“达标小庆祝”开关。
- 左键打卡后菜单栏图标会短暂高亮。
- 首次达标时显示 1.2 秒自动关闭的小庆祝 popover。
- 右键菜单新增“上次：刚刚 / X 分钟前 / 今日未打卡”。
- CoreChecks 增加提醒规则、旧数据兼容、打卡时间和跨天提醒状态校验。

### 优化

- 只有存在已启用提醒的场景时，才请求或提示通知权限。
- 默认提醒关闭，默认动效和小庆祝开启，保持菜单栏工具的轻量使用方式。

## 0.2.1 - 多场景缺陷修复

### 修复

- 修复菜单栏图标未按设置页选择的 SF Symbols 正常展示的问题。
- 修复普通保存设置时反复触发开机自启注册/取消注册，导致出现开机自启失败提示的问题。

### 优化

- 菜单栏主图标和右键菜单场景切换图标统一使用同一套 SF Symbols 渲染路径。
- 开机自启只有在用户实际修改开关时才调用系统接口，普通场景保存不再受本地打包权限影响。

## 0.2.0 - 多场景次数型打卡

### 新增

- 支持多个次数型打卡场景。
- 设置窗口新增场景管理，可新增、编辑、停用场景。
- 场景支持内置 SF Symbols 图标、主题色、启用状态和固定到菜单栏。
- 菜单栏新增场景展示模式：当前场景 / 固定场景多图标。
- 右键菜单支持用图标切换场景，并展示各场景今日进度。
- 支持 `⌥⌘1-9` 按场景排序快速切换前 9 个启用场景。

### 变更

- 数据层从单事项升级为多场景模型，旧事项和旧今日状态会迁移为默认场景。
- 今日状态和历史记录按场景 ID 隔离，跨天时按启用场景分别归档。
- 当月水位日期表展示当前菜单对应场景的历史进度。

## 0.1.3 - 菜单栏与设置体验优化

### 新增

- 增加菜单栏显示模式设置，支持“只显示图标”和“图标 + 数字”。

- 新增旧版设置数据兼容逻辑，避免新增显示模式字段后导致原设置回退默认值。

## 0.1.2 - 打卡音效修复

## 0.1.0 - MVP 可运行版本

- 菜单栏常驻应用「点一点」。
- 单事项次数型打卡，默认事项“喝水”。

- 右键菜单展示事项名称、今日次数、每日目标。

- 跨天自动归档前一天历史并重置今日次数。
- 自绘进度环图标。

### 修复

- 修复 App 启动后菜单栏项未显示的问题。

- 开机自启在 SwiftPM 本地打包场景下仍需进一步验证。
