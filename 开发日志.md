## 2026-05-21 23:20 - MVP 初版落地

### 目标

根据 `MVP功能清单.md` 实现「点一点」菜单栏次数打卡工具的第一版可运行原型。

### 调整内容

- 创建 SwiftPM 工程，应用模块为 `DianYiDian`，核心业务模块为 `DianYiDianCore`。
- 实现单事项次数型打卡：默认事项“喝水”，默认目标 8 次，默认初始次数 0。
- 实现菜单栏常驻、左键加 1、右键菜单、设置窗口、本地存储和跨天归档。
- 增加本地 `.app` 打包脚本，生成 `点一点.app`。
- 增加核心业务校验入口 `DianYiDianCoreChecks`。

### 验证结果

- `swift build` 通过。
- `swift run DianYiDianCoreChecks` 通过。
- `sh Scripts/package-app.sh` 通过。
- 本地生成的 `点一点.app` 可以启动为进程。

### 发现的问题

- 当前机器没有完整 Xcode，暂时无法做标准 Xcode Archive。
- 首次验证时只确认进程启动，没有确认菜单栏项真实可见，验证标准不够完整。

### 下一步

- 增加针对菜单栏可见性的手动验收步骤。
- 后续补完整 Xcode 工程或 Archive 流程。

## 2026-05-21 23:35 - 修复菜单栏图标不可见

### 目标

修复启动 App 后用户在菜单栏看不到应用图标的问题。

### 调整内容

- 将 AppKit 启动方式从隐式 `@main AppDelegate` 改为显式 `main.swift` 启动。
- 在 `main.swift` 中创建 `NSApplication.shared`，设置 `AppDelegate`，并运行事件循环。
- 将菜单栏项改为固定宽度。
- 菜单栏项显示从纯图标调整为“进度图标 + 次数/目标”，例如 `1/8`。

### 验证结果

- `swift build` 通过。
- `swift run DianYiDianCoreChecks` 通过。
- `sh Scripts/package-app.sh` 通过。
- 启动 `点一点.app` 后，系统辅助功能可读到 `DianYiDian menu bars: 1 -> 1/8`。
- 截图确认菜单栏中可见进度图标和 `1/8` 文本。

### 发现的问题

- 在多显示器或菜单栏空间紧张场景下，菜单栏项可能出现在非当前截图区域，需要手动确认实际显示位置。
- 右键菜单还需要进一步做稳定性验收。

### 下一步

- 补充菜单栏左键、右键、设置窗口的手动验收清单。
- 继续优化图标可读性和菜单交互手感。
