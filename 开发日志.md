## 2026-05-23 22:57 - 0.4.1 Liquid Glass 体验修复

### 目标

修复 0.4.0 后用户反馈的三个问题：右键菜单栏浮层顶部被裁切；提醒方式时间只能用步进器，不能手动输入；点击一个场景打卡时多个菜单栏场景都有响应感。

### 调整内容

- 将右键浮层从 `NSPopover` 路径改回稳定的 `NSMenu` 弹出机制，并用单个自定义 `NSMenuItem.view` 嵌入 SwiftUI 玻璃菜单内容。
- 压缩右键菜单内容尺寸，降低在菜单栏附近被系统裁切的概率。
- `StatusBarController` 收到自身发出的打卡变更通知时不再二次全量刷新。
- 打卡动效和达标动效只刷新当前场景对应的菜单栏项。
- 提醒设置中的间隔分钟、固定小时和固定分钟改为可手动输入的数字框，并保留范围限制。

### 验证结果

- `swift build` 通过。
- `swift run DianYiDianCoreChecks` 通过。
- `sh Scripts/package-app.sh` 通过。
- 打包后重新启动 `点一点.app`，菜单栏项可见。
- 左键点击第一个菜单栏场景后，计数从 `12/8` 更新为 `13/8`，第二个场景保持 `11/1` 不变。
- 右键菜单截图确认浮层完整显示在菜单栏下方，顶部内容不再被裁切。

### 发现的问题

- 玻璃菜单内容目前作为自定义 `NSMenuItem.view` 承载，视觉上仍保留一点系统菜单外壳；这是为了优先保证菜单栏右键定位稳定。
- 后续如果要完全无系统菜单外壳，需要再做独立浮窗方案并补事件命中自动化测试。

### 下一步

- 增加菜单栏多图标点击的自动化验收脚本。
- 增加设置页提醒时间输入保存的自动化验收脚本。

## 2026-05-23 18:35 - 0.3.0 轻提醒与打卡反馈

### 目标

完成 0.3.0：在不增加复杂统计和重入口的前提下，为主动打卡增加轻量反馈，为容易忘记的场景增加本地轻提醒。

- 左键打卡后菜单栏图标短暂高亮；首次达标时显示 1.2 秒自动关闭的小庆祝 popover。
- 右键菜单新增“上次：刚刚 / X 分钟前 / 今日未打卡”。
- 更新 `DianYiDianCoreChecks`，覆盖提醒规则、旧数据兼容、打卡时间更新和跨天提醒状态重置。

### 验证结果

- `swift build` 通过。
- `swift run DianYiDianCoreChecks` 通过。
- `sh Scripts/package-app.sh` 通过。
- 重新启动 `点一点.app` 后，系统辅助功能可读到菜单栏项 `2/8, 0/1`。
- 左键点击菜单栏项后，计数能从 `1/8` 更新为 `2/8`。
- 右键菜单能打开，菜单内可读到“上次：2 分钟前”。
- 设置窗口可见“打卡动效”“达标小庆祝”和“提醒”区域。
- 默认“不提醒”时，设置窗口不再显示通知权限提示。

### 发现的问题

- 通知真实弹出、音效听感和动效观感仍属于人工体验项，当前只完成本机可见路径和构建校验。
- 由于本机通知权限已被系统拒绝过，开启提醒后的权限提示需要在系统设置中恢复权限后再做完整体验确认。

### 下一步

- 增加 App UI 自动化验收脚本，覆盖菜单栏右键、设置页保存和提醒配置持久化。
- 继续保持 0.3.x 克制迭代，不扩展独立统计页和复杂模板库。
