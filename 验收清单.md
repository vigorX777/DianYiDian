# 验收清单

本文件用于记录「点一点」每次开发后的固定验收项。后续功能开发完成后，应在 `开发日志.md` 中记录本清单的执行结果。

## 1. 基础构建验收

- [ ] `swift build` 可以通过。
- [ ] `swift run DianYiDianCoreChecks` 可以通过。
- [ ] `sh Scripts/package-app.sh` 可以生成 `点一点.app`。
- [ ] 生成的 `点一点.app` 可以启动。
- [ ] 启动后进程名为 `DianYiDian`。

## 2. 菜单栏显示验收

- [ ] App 启动后只显示在菜单栏，不显示 Dock 图标。
- [ ] 菜单栏项可见。
- [ ] 菜单栏项显示进度图标。
- [ ] 菜单栏项显示今日次数和每日目标，例如 `1/8`。
- [ ] 深色菜单栏背景下可识别。
- [ ] 浅色菜单栏背景下可识别。
- [ ] 菜单栏空间紧张时仍能确认 App 存在。

## 3. 左键打卡验收

- [ ] 左键点击菜单栏项后，今日次数加 1。
- [ ] 左键点击不会弹出菜单或窗口。
- [ ] 今日次数变化后，菜单栏文本同步刷新。
- [ ] 今日次数变化后，进度图标同步刷新。
- [ ] 单次点击打卡音效能播放。
- [ ] 关闭“打卡成功提示”后，单次点击音效不播放。

## 4. 目标完成音效验收

- [ ] 今日次数首次达到每日目标时，播放目标完成音效。
- [ ] 达标后继续点击，不重复播放目标完成音效。
- [ ] 关闭“达到每日目标时提醒”后，目标完成音效不播放。
- [ ] 修改每日目标后，目标完成判断按新目标生效。

## 5. 右键菜单验收

- [ ] 右键点击菜单栏项后，菜单能稳定打开。
- [ ] 菜单显示当前事项名称。
- [ ] 菜单显示今日次数和每日目标。
- [ ] 菜单中的“打卡”可以让今日次数加 1。
- [ ] 菜单中的“撤销”可以撤销最近一次打卡。
- [ ] 今日次数为 0 或没有可撤销打卡时，“撤销”不可用。
- [ ] 菜单中的“重置”可以将今日次数重置为今日初始次数。
- [ ] 菜单中的“设置”可以打开设置窗口。
- [ ] 菜单中的“退出点一点”可以退出 App。

## 6. 当月历史日期表验收

- [ ] 右键菜单中显示一个小型当月日期表。
- [ ] 日期表展示当前月份所有日期。
- [ ] 日期表能区分今天、过去日期和未来日期。
- [ ] 每个日期点能显示当天打卡状态。
- [ ] 每个日期点使用水位图效果表达完成比例。
- [ ] 未打卡日期显示空水位。
- [ ] 部分完成日期显示对应比例水位。
- [ ] 达标日期显示满水位或完成态。
- [ ] 超过目标日期仍按满水位显示。
- [ ] 当月历史数据来自本地历史记录和今日状态。

## 7. 设置窗口验收

- [ ] 设置窗口可以打开。
- [ ] 开机自启开关可以保存状态。
- [ ] 打卡成功提示开关可以保存状态。
- [ ] 达到每日目标提醒开关可以保存状态。
- [ ] 事项名称可以保存并立即同步到菜单。
- [ ] 每日目标次数限制在 1 到 99。
- [ ] 今日初始次数限制在 0 到 99。
- [ ] 图标样式可以切换。
- [ ] 保存后菜单栏图标和文本立即刷新。
- [ ] 关闭再打开设置窗口时，表单显示上次保存的值。

## 8. 数据与跨天验收

- [ ] App 重启后保留事项配置。
- [ ] App 重启后保留今日次数。
- [ ] 跨天后将前一天记录写入历史。
- [ ] 跨天后今日次数重置为今日初始次数。
- [ ] 手动重置只影响今日次数，不删除历史记录。
- [ ] 历史记录文件为空或损坏时，App 不崩溃。

## 9. Xcode 与正式分发验收

- [ ] 存在完整 Xcode 工程或可稳定生成 Xcode 工程。
- [ ] Bundle ID 为 `com.vigor.dianyidian`。
- [ ] App 显示名为“点一点”。
- [ ] `LSUIElement` 生效，App 不显示 Dock 图标。
- [ ] App 有正式图标。
- [ ] 可以完成 Archive。
- [ ] 可以完成本地签名。
- [ ] 导出的 App 可以在本机安装并启动。
- [ ] 发布前版本号和构建号已更新。

## 10. 文档与日志验收

- [ ] 本次开发前已查看 `任务看板.md`。
- [ ] 本次开发后已更新 `开发日志.md`。
- [ ] 如果产生产品或技术方向变化，已更新 `决策记录.md`。
- [ ] 如果影响版本行为，已更新 `版本记录.md`。
- [ ] 本次开发已形成 git 提交。
